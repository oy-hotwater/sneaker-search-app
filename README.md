# sneaker-search-app

## 概要

本プロジェクトは、深層学習モデルによる特徴量抽出と近似最近傍探索を組み合わせた、スニーカーの類似画像検索システムです。

ユーザーが入力したスニーカーの画像に対し、データベース内から視覚的に類似するモデルを高速に検索・提示します。Jupyter Notebookでの検証を経て、実運用を想定したWebアプリケーションへの拡張を進めています。

## システムアーキテクチャ（全体像）
本システムは、保守性とスケーラビリティを考慮し、以下の技術スタックを用いて構築・コンテナ化を行います。

1. フロントエンド: Streamlit
   - ユーザーからの画像アップロード受け付け、および検索結果（類似画像群）のUI描画を担当します。
2. バックエンドAPI: FastAPI
   - クライアントからのリクエストを受信し、推論モデルと検索エンジンの呼び出しを制御します。
3. 機械学習・検索基盤: TensorFlow / Faiss
   - TensorFlowを用いた画像のベクトル特徴量抽出と、メタ社のFaissインデックスを用いたミリ秒単位の高速検索を実行します。
4. インフラストラクチャ: Docker / クラウドホスティング
   - Dockerによるコンテナ化を行い、環境依存を排除します。Hugging Face Spaces等のクラウド環境へデプロイし、常時アクセス可能な状態を構築します。

## 主要な機能と評価ポイント
- 高速なベクトル検索: Faissの導入により、単なる特徴量比較ではなく、大規模データに対しても低遅延で検索可能なプロダクションレディな設計を採用しています。
- 堅牢なパイプライン: 画像読み込み時の破損ファイル事前チェックや、tf.data APIを用いた非同期ストリーミング処理により、システムエラーを未然に防ぐハンドリングを実装しています。
- セキュアな設計: APIキーなどの機密情報を環境変数やシークレットとして分離し、コードベースからの漏洩を防止しています。

## 注意事項（デプロイ環境について）
本アプリケーションは無料枠のクラウドサーバーを活用して運用する予定です。リソース最適化の仕様上、一定時間アクセスがない場合はサーバーがスリープ状態に移行します。
そのため、初回のアクセスおよび最初の検索実行時には、コンテナの起動処理により約30秒〜1分程度の遅延が発生する場合があります。あらかじめご了承ください。
